Bug修复

弃用

Debian Jessie和Raspbian Stretch被弃用

Debian Jessie和Raspbian Stretch在2020年6月停产。GitLab13.3是这些发行版的最后一个受支持版本。请访问“ 不推荐使用的操作系统” 页面以获取更多信息。

弃用日期： GitLab 13.3

弃用容器注册表日志格式化程序

目前，GitLab支持针对应用程序日志的text / json / logstash日志格式和针对访问日志的text / json / combined。我们将弃用logstash和组合格式，仅使用两个选项（文本（用于开发）和json）统一应用程序和访问日志的格式化程序。

弃用日期： 2021年1月22日

弃用Container Registry日志挂钩

Container Registry支持日志挂钩，目前仅可用于电子邮件通知。

如今，基于日志条目的警报通常由单独的工具处理。据我们所知，我们的用户都没有依赖此功能，GitLab也未使用它。此功能的实现与基础日志库紧密结合，这是我们在不影响可用功能的情况下切换依赖关系的能力的限制。

为了简化注册表功能和配置，我们将放弃对日志记录挂钩的支持。

弃用日期： 2021年1月22日

弃用Container Registry maxidle和maxactive Redis池设置

我们当前为Redis连接池公开的一些配置设置与基础Redis客户端绑定，并且在替代库中没有等效设置。在我们开始改进Redis集成（例如增加对Sentinel的支持）的工作时，我们决定开始着手用功能更丰富，支持更好的替代方法来替换当前的Redis客户端依赖项。为此，我们需要替换绑定到当前客户端库的当前Redis池配置设置。

我们打算删除redis.pool.maxidle和redis.pool.maxactive设置，并添加redis.pool.size（最大连接数），redis.pool.miniddle（最小空闲连接数）和redis.pool.maxlifetime（可以重新使用连接的最大时间）。

弃用日期： 2021年1月22日

弃用Container Registry代理穿透缓存

在最近对GitLab Admin进行调查之后，我们决定不弃用此功能。

本proxy部分允许将Container Registry设置为上游存储库的本地镜像。

弃用日期： 无限期推迟

弃用Bugsnag的Container Registry支持

Bugsnag是容器注册表支持的错误报告服务之一。据我们所知，没有用户依赖此服务，在GitLab我们使用Sentry。为了简化和整合支持的错误报告服务，我们打算添加对Sentry的支持，并删除对Bugsnag的支持。

弃用日期： 2021年1月22日

弃用对NewRelic的Container Registry支持

NewRelic是容器注册表支持的错误报告服务之一。据我们所知，没有用户依赖此服务，在GitLab我们使用Sentry。为了简化和整合支持的错误报告服务，我们打算添加对Sentry的支持，并删除对NewRelic的支持。

弃用日期： 2021年1月22日

在秘密检测模板中弃用SAST_EXCLUDED_PATHS

为了使托管CI模板保持一致和简化，我们SAST_EXCLUDED_PATHS将Secret Detection CI模板中的变量重命名为SECRET_DETECTION_EXCLUDED_PATHS。这将始终将所有Secret Detection CI模板变量命名为以开头SECRET_DETECTION_。请注意，这SAST_EXCLUDED_PATHS仍然是SAST CI模板中的有效变量。此更改仅影响秘密检测CI模板。为了过渡，如果你使用一个定制的秘密检测CI模板，更新您的变量名模板。如果您使用Auto DevOps或继承GitLab的托管安全模板，则无需执行任何操作。

弃用日期： 2020年8月22日

不推荐使用地理外部数据包装器设置

从GitLab 13.2开始，Geo不需要为数据复制配置外部数据包装程序（FDW）。结果，以下配置设置/etc/gitlab/gitlab.rb已弃用，并将在14.0中删除：

  • geo_secondary['db_fdw']
  • geo_postgresql['fdw_external_user']
  • geo_postgresql['fdw_external_password']
  • gitlab-_rails['geo_migrated_local_files_clean_up_worker_cron']

弃用日期： GitLab 14.0

WIP（进行中的）合并请求术语已弃用

我们将合并请求的WIP（在进行中）术语重命名为“草稿”，因为它更具包容性且不言自明。现在不建议使用WIP术语。我们将在下一个主要的GitLab版本（14.0）中支持它的使用，然后将其删除。

弃用日期： 2021年4月17日

清除

地理耙任务以刷新已删除的外部表

从GitLab 13.2开始，Geo不需要为数据复制配置外部数据包装程序（FDW）。结果，geo:db:refresh_foreign_tables不再需要瑞克任务，并且在此版本中已将其删除。

删除日期： GitLab 13.3

删除SAST秘密检测作业，转而使用新模板

从GitLab 13.1开始，秘密检测CI / CD配置设置现在在单独的GitLab提供的模板中提供，并作为新的安全扫描类型运行。现在，Auto DevOps中还包含了这个新的秘密检测模板，以便于设置。

在13.4中，我们将删除旧的SASTsecrets-sast作业定义，并且如果您尚未切换到新的“秘密检测”模板，则不会继续扫描秘密。您可以通过添加新模板轻松地进行过渡。

在升级到GitLab 13.4之前，我们建议您将新的Secret Detection模板添加到gitlab-ci.yml文件中，然后从文件中的SAST配置模板中删除旧的SASTsecrets-sast作业定义。我们制作了一个视频，指导您完成过渡到此新模板的过程。SAST.gitlab-ci.yml

您可以关注清除问题，以获取更多详细信息。
